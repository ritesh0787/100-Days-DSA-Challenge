## üí° Study Tips

1. **Understand Before Memorizing**: Focus on understanding concepts rather than memorizing solutions
2. **Practice Regularly**: Solve at least 2-3 problems daily
3. **Time Yourself**: Practice with time constraints to simulate interviews
4. **Discuss with Peers**: Join study groups or online communities
5. **Review Mistakes**: Learn from failed attempts and wrong submissions
6. **Mock Interviews**: Practice with friends or on platforms like Pramp

## üéØ Problem-Solving Approach

### Step 1: Understand the Problem
- Read the problem statement carefully
- Identify inputs, outputs, and constraints
- Ask clarifying questions

### Step 2: Plan the Solution
- Think of edge cases
- Consider different approaches
- Analyze time and space complexity

### Step 3: Write Clean Code
- Use meaningful variable names
- Add comments for complex logic
- Follow language conventions

### Step 4: Test Thoroughly
- Test with sample inputs
- Check edge cases
- Verify complexity

### Step 5: Optimize
- Can you do better?
- Trade-offs between time and space
- Discuss with interviewer

## üåü Important Patterns

1. **Two Pointers**: Array problems, LinkedList
2. **Sliding Window**: Substring problems
3. **Fast & Slow Pointers**: Cycle detection
4. **Merge Intervals**: Overlapping intervals
5. **Cyclic Sort**: Array with numbers in range
6. **In-place Reversal**: LinkedList reversal
7. **Tree BFS**: Level-order traversal
8. **Tree DFS**: Tree traversal problems
9. **Two Heaps**: Median finding
10. **Subsets**: Permutations and combinations
11. **Modified Binary Search**: Sorted arrays
12. **Top K Elements**: Heap problems
13. **K-way Merge**: Multiple sorted arrays
14. **Dynamic Programming**: Optimization problems
15. **Topological Sort**: Graph ordering
